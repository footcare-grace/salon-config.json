# フットケアサロン集客支援ツール

フットケアサロンの SNS 経由の新規顧客獲得を支援する、**API 不要・完全静的**の Web ツールです。
サロンの基本情報を入力すると、以下 4 種類の AI 用プロンプトが自動生成されます。生成されたプロンプトをコピーして、Claude や Gemini などの対話型 AI に貼り付けるだけで使えます。

| # | プロンプト | 内容 |
|---|-----------|------|
| ① | リサーチチーム用 | ペルソナ設計・悩みの言語化・地域競合分析と差別化ポジショニング |
| ② | コンテンツ制作チーム用 | Threads / note 投稿案、リール構成案(台本・カット割り)、週次投稿カレンダー |
| ③ | マーケティングチーム用 | SNS→LINE→予約のファネル設計、LINE 登録特典、初回予約ハードル低減策 |
| ④ | 統合ディレクター用 | ①〜③の出力を貼り付けて「今週やるべきこと」実行プランに変換(優先順位・担当割りつき) |

- HTML / CSS / JavaScript のみで動作し、**サーバー処理・API 通信は一切ありません**
- 入力内容が外部に送信されることはありません(手動コピペ前提)
- サロン情報は `salon-config.json` に分離されており、**差し替えるだけで別サロンでも再利用できます**

## ディレクトリ構成

```
/
├── index.html         … 画面本体
├── style.css          … スタイル
├── script.js          … プロンプト生成ロジック
├── salon-config.json  … サロン基本情報(要編集)
└── README.md
```

## 使い方

1. このページを開くと、`salon-config.json` の内容がフォームに読み込まれます
2. 必要に応じてフォーム上で情報を修正し、**「この内容でプロンプトを更新」** を押します
   - 修正内容はお使いのブラウザにのみ一時保存されます(localStorage)
   - **「salon-config.json の内容に戻す」** でいつでも設定ファイルの状態に戻せます
3. ①〜④のタブを開き、各プロンプトの **「コピー」** ボタンでクリップボードにコピーします
4. Claude / Gemini などに貼り付けて実行します

### おすすめの実行順

```
① リサーチ
   ↓ コピーしてClaude/Geminiで実行 → 回答が返ってくる
   ↓ その回答を「④統合ディレクター」タブの入力欄に貼り付ける
     → ②③のプロンプト内、④のプロンプト内に自動で反映される
② コンテンツ制作 をコピーしてAIで実行 → 回答を④の入力欄に貼り付け
③ マーケティング をコピーしてAIで実行 → 回答を④の入力欄に貼り付け
④ 統合ディレクター をコピーしてAIで実行 → 今週の実行プランが完成
```

**ポイント:** ④タブには①②③それぞれの回答を貼り付ける入力欄があります。ここに貼り付けると、②③④のプロンプト本文にある「貼り付け欄」へ自動的に反映されます(このページ内でのコピー&ペースト作業は不要です)。貼り付け内容はブラウザに一時保存されるため、再訪時も残ります。

## salon-config.json の編集方法(自分のサロン用にカスタマイズする)

`salon-config.json` をテキストエディタで開き、各項目を自分のサロンの情報に書き換えて保存します。

```json
{
  "salonName": "サロン名",
  "strengths": [
    "強み1",
    "強み2"
  ],
  "differentiators": [
    "差別化ポイント1"
  ],
  "targetArea": "対象エリア(例:○○市△△区)",
  "priceRange": "価格帯(例:初回体験 5,500円〜)",
  "lineAccountName": "公式LINEのアカウント名(例:@your_salon)",
  "otherNotes": "客層や営業時間などの補足メモ"
}
```

編集時の注意:

- `strengths` と `differentiators` は配列です。項目を `"..."` で囲み、項目の間はカンマ `,` で区切ってください(最後の項目の後ろにカンマは不要)
- 文字コードは **UTF-8** で保存してください
- JSON の文法エラーがあると読み込みに失敗します(その場合もフォームへの手入力は可能です)

> **補足:** ローカルで `index.html` をダブルクリックして開いた場合、ブラウザのセキュリティ制約により `salon-config.json` の自動読み込みが行われないことがあります。その場合はフォームに直接入力するか、後述の GitHub Pages で公開してご利用ください。

## GitHub Pages での公開手順

1. GitHub で新規リポジトリを作成します(例:`footcare-salon-tool`)
2. このフォルダ内の 5 ファイルをリポジトリのルートにアップロード(またはプッシュ)します
3. リポジトリの **Settings → Pages** を開きます
4. **Source** で `Deploy from a branch` を選択し、Branch に `main` / フォルダ `/ (root)` を指定して **Save**
5. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` でツールが公開されます

サロン情報を更新したいときは、`salon-config.json` を編集してコミットするだけで反映されます。

## 再配布について

本ツールはサロン情報をコードから分離した設計のため、`salon-config.json` を差し替えるだけで、第三者が自分のサロン用ツールとしてそのまま利用・再配布できます。フォーク後に設定ファイルを編集し、各自の GitHub Pages で公開してください。

## 注意事項

- 生成されるプロンプトは AI への指示文であり、AI の出力内容の正確性・法令適合性(医療広告ガイドライン・薬機法等)は利用者ご自身でご確認ください
- 個人情報や機微な情報をフォームに入力する必要はありません
